This is a C++ project using the \href{https://www.sfml-dev.org/tutorials/3.0/}{\texttt{ SFML}} library for graphics and input; CMake for building; \href{https://github.com/google/googletest}{\texttt{ Google\+Test}} for testing; and \href{https://doxygen.nl/}{\texttt{ Doxygen}} for documentation.

Code is split in two in the {\ttfamily src/} folder. Within {\ttfamily src/lib} are all the files students participating in the competition do not need to worry about. While the files in the root of {\ttfamily src/} are ones the students will worry about.\hypertarget{md_documentation_2CONTRIBUTING_autotoc_md14}{}\doxysubsection{\texorpdfstring{The Main Program}{The Main Program}}\label{md_documentation_2CONTRIBUTING_autotoc_md14}
If you open {\ttfamily src/main.\+cpp}, you\textquotesingle{}ll notice that it essentially manages the window, draws the graphics, and keeps state around the the {\ttfamily compute\+\_\+game\+\_\+logic} function. As you can guess, {\ttfamily compute\+\_\+game\+\_\+logic} does all the heavy lifting for deciding how snakes move, spawning fruits, etc.\hypertarget{md_documentation_2CONTRIBUTING_autotoc_md15}{}\doxysection{\texorpdfstring{Workflow}{Workflow}}\label{md_documentation_2CONTRIBUTING_autotoc_md15}
To contribute, a typical workflow is used\+: Fork, branch, and then make a pull request to the main branch. Joel will review the request, make feedback as necessary, and approve.

Make sure to install and run your code through \href{https://clang.llvm.org/docs/ClangFormat.html}{\texttt{ Clang\+Format}}.\hypertarget{md_documentation_2CONTRIBUTING_autotoc_md16}{}\doxysection{\texorpdfstring{Running The Project}{Running The Project}}\label{md_documentation_2CONTRIBUTING_autotoc_md16}
See README.\+md for instructions on compiling and running.\hypertarget{md_documentation_2CONTRIBUTING_autotoc_md17}{}\doxysection{\texorpdfstring{Tests}{Tests}}\label{md_documentation_2CONTRIBUTING_autotoc_md17}
We use Google\+Test as our testing framework. To run tests, build the project like usual\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{cmake\ -\/-\/build\ build}

\end{DoxyCode}


And then run the resulting test binary\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{./build/bin/main\_tests}

\end{DoxyCode}

Documentation blocks are written in comments with three slashes. Every element you want to document is preceded by these blocks. If we take a closer look at one of these\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{}
\DoxyCodeLine{\mbox{\hyperlink{cell_8h_a0133c02dfc35ffbaf07ad1a587dac4d1}{Cell}}\ get(\textcolor{keywordtype}{int}\ x,\ \textcolor{keywordtype}{int}\ y)\ \textcolor{keyword}{const};}

\end{DoxyCode}


Special commands in Doxygen are prepended with a backslash. The {\ttfamily \textbackslash{}brief} command and the text that follows it is the brief description of that element that will be rendered at the top. The rest of the doc block (separated with a newline), will be rendered below the brief.

Something else you may notice are doc blocks surrounded with \`{}\`{}. These tell Doxygen what {\itshape not} to include in the final generated documentation. You\textquotesingle{}ll see these "{}conditionals"{} around items that students don\textquotesingle{}t need to worry about when creating a bot.\hypertarget{md_documentation_2CONTRIBUTING_autotoc_md20}{}\doxysubsection{\texorpdfstring{Building}{Building}}\label{md_documentation_2CONTRIBUTING_autotoc_md20}
If modifying documentation and you need to see your changes, you can build locally.

\item Run {\ttfamily doxygen}
\item And then with the web browser of your choice, open up {\ttfamily ./docs/html/index.html} 
\end{DoxyItemize}
